# **دليل تطوير وبرمجة الوكلاء في نظام Agent Mesh**

يوفر هذا الدليل شرحاً تفصيلياً لكيفية إضافة وكلاء جدد للنظام وتكليفهم بمهام محددة باستخدام بنية البرمجة كائنية التوجه (OOP) المعتمدة في المشروع، مع استعراض أهم الخيارات المتاحة للتحكم في سلوكهم ومظهرهم.

## **1\. إضافة وكيل جديد (Adding an Agent)**

يتم إضافة الوكلاء في مرحلة التهيئة الأولية للنظام لضمان ثبات الهوية في الذاكرة ومنع حدوث التكرار عند إعادة طلب المهام. يتم ذلك عبر استدعاء الدالة create\_agent من خلال مدير الشبكة mesh.

mesh.create\_agent(name, role, gender, color, agent\_id)

### **الخيارات والمعايير المتاحة:**

| الخيار | الوصف التقني | أمثلة مقترحة   |
| ----- | ----- | ----- |
| **name** | الاسم التعريفي للوكيل الذي يظهر للمستخدم في الواجهة الرسومية. | MAX, المنسق، المحلل |
| **role** | التخصص الوظيفي للوكيل، ويظهر تحت الاسم بخط أصغر. | System Core, Security Agent |
| **gender** | تحديد الجنس لاختيار الـ Avatar المناسب (ذكر أو أنثى). | male, female |
| **color** | كود اللون (Hex Code) الذي سيميز عناصر الوكيل وبطاقته. | \#6366f1 (أزرق), \#10b981 (أخضر) |
| **agent\_id** | **(هام جداً)** المعرف الفريد الذي يربط الكود بالواجهة الرسومية. | core\_1, data\_processor\_v2 |

## **2\. إسناد الوظائف والمهام (Task Management)**

لجعل الوكيل ينفذ وظيفة معينة، نستخدم نظام "مدير السياق" with mesh.task(agent\_id). هذا الأسلوب يضمن إدارة الحالة (State) بشكل آلي.

### **كيفية عمل مدير السياق:**

* **بداية المهمة:** يتم إرسال أمر للواجهة لتحويل حالة الوكيل إلى **Working** (تظهر حركة النبض والرسوم النشطة).  
* **تنفيذ الكود:** يتم تشغيل أي منطق برمي داخل الكتلة (مثل معالجة نصوص أو جلب بيانات).  
* **نهاية المهمة:** فور الخروج من الكتلة، يعود الوكيل تلقائياً إلى حالة **Sleeping**.

\# مثال لإسناد مهمة  
with mesh.task("core\_1"):  
    print("الوكيل بدأ العمل...")  
    time.sleep(3)  \# محاكاة لعملية معالجة

## **3\. التفاعل والربط البصري (Interactions)**

يمكنك جعل الوكلاء يتفاعلون بصرياً من خلال رسم خطوط نقل بيانات بين معرفاتهم عبر الدالة connect\_visual:

mesh.connect\_visual(source\_id="core\_1", target\_id="data\_1", log\_msg="نقل البيانات للتحليل")

## **4\. نصائح برمجية هامة**

1. **تجنب التكرار:** لا تقم بإنشاء الوكلاء داخل دوال المهام المتكررة، بل في الدالة الخاصة بالتهيئة (Initialization).  
2. **استخدام الألوان:** استخدم ألواناً متباينة لتسهيل التمييز البصري بين تخصصات الوكلاء.  
3. **السجلات (Logs):** احرص على إرسال رسائل سجل واضحة عبر mesh.log لمتابعة سير العملية في اللوحة الجانبية.
